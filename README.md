# Angular 助手扩展

Angular 助手扩展是一个为 Visual Studio Code 设计的工具，旨在提高 AngularJS (1.x) 开发的效率。它提供了强大的定义查找功能，支持在 HTML 和 JavaScript 文件之间进行跳转。

## 系统要求

- Visual Studio Code 版本 1.93.0 或更高

## 功能

1. **定义查找**：
   - 在 HTML 文件中查找 AngularJS 表达式、指令和控制器的定义
   - 在 JavaScript 文件中查找 HTML 模板中使用的变量和函数

2. **跨文件跳转**：
   - 从 HTML 模板跳转到相关的 JavaScript 定义
   - 从 JavaScript 文件跳转到 HTML 模板中的使用位置

3. **实时解析**：
   - 在文件保存时自动更新解析结果
   - 优先解析当前打开的文件，提高响应速度

4. **性能优化**：
   - 使用 LRU 缓存来存储解析结果，提高查找速度
   - 批量处理文件解析，避免阻塞 UI
   - 支持并发解析，提高大型项目的解析速度

5. **配置选项**：
   - 可自定义缓存大小和缓存过期时间
   - 可配置需要排除的文件模式
   - 可启用性能日志记录和普通日志记录

## 使用方法

1. 确保你的 Visual Studio Code 版本不低于 1.93.0。
2. 在 VS Code 中安装 Angular 助手扩展。
3. 打开一个 AngularJS (1.x) 项目。
4. 在 HTML 或 JavaScript 文件中，将光标放在要查找定义的元素上。
5. 使用 "转到定义"（通常是 F12 键）或右键菜单中的 "转到定义" 选项。
6. 扩展将自动跳转到相应的定义位置。

## 日志和调试

Angular 助手扩展提供了详细的日志记录，可以帮助你了解扩展的工作状态和诊断潜在问题。

### 查看日志

1. 在 VS Code 底部面板中，点击 "输出" 选项卡（或使用快捷键 Ctrl+Shift+U / Cmd+Shift+U）。
2. 在输出面板顶部的下拉菜单中，选择 "Angular Helper"。
3. 你将看到扩展的详细运行日志，包括文件解析状态、错误信息等。

这些日志对于理解扩展的工作过程和诊断问题非常有用。如果你在使用扩展时遇到任何问题，请查看这些日志，它们可能会提供有价值的调试信息。

## 性能日志

Angular 助手扩展还提供了一个命令来查看性能日志，这对于调试和优化扩展的性能非常有用。

### 查看性能日志

1. 确保在设置中启用了性能日志记录（`angularHelper.enablePerformanceLogging` 设置为 `true`）。
2. 使用扩展一段时间，以收集足够的性能数据。
3. 打开 VS Code 的命令面板（Windows/Linux: Ctrl+Shift+P, macOS: Cmd+Shift+P）。
4. 输入并选择 "Angular Helper: Output Performance Report"。
5. 性能报告将显示在 VS Code 的输出面板中。

### 性能报告解读

性能报告包含以下信息：

- 各种操作（如缓存命中、定义查找等）的平均执行时间。
- 每种操作的执行次数。

这些信息可以帮助你了解扩展的性能瓶颈，并决定是否需要调整配置或优化工作流程。

注意：频繁使用性能日志可能会略微影响扩展的整体性能。在正常使用时，建议保持性能日志禁用。

## 配置选项

在 VS Code 的设置中，你可以自定义以下选项来优化 Angular 助手扩展的性能和行为：

### `angularHelper.definitionCacheSize`
- **描述**：定义缓存的大小，即可以同时缓存多少个定义位置。
- **默认值**：100
- **说明**：增加这个值可以提高频繁访问的定义的查找速度，但会占用更多内存。对于大型项目，可以考虑增加这个值。

### `angularDefinitionProvider.cacheTTL`
- **描述**：缓存的生存时间，单位为毫秒。
- **默认值**：3600000（1小时）
- **说明**：这个设置决定了缓存中的项目多久后会失效。增加这个值可以提高性能，但可能导致看到过时的定义。如果你的项目结构变化频繁，可以考虑减小这个值。

### `angularHelper.enablePerformanceLogging`
- **描述**：是否启用性能日志记录。
- **默认值**：false
- **说明**：启用此选项后，扩展会在输出面板中记录各种操作的性能数据。这对于调试和优化很有用，但在正常使用时建议保持禁用状态。

### `angularDefinitionProvider.excludePatterns`
- **描述**：要排除的文件模式数组。
- **默认值**：[".git/**", ".history/**", ".idea/**", ".vscode/**", "doc/**"]
- **说明**：这里可以指定不需要解析的文件或目录。例如，你可以排除测试文件或第三方库文件夹。正确配置这个选项可以显著提高初始化速度和减少内存使用。

### `angularDefinitionProvider.maxConcurrentParsing`
- **描述**：最大并发解析文件数。
- **默认值**：5
- **说明**：这个设置控制同时可以解析多少个文件。增加这个值可以加快初始化速度，但也会增加 CPU 和内存的使用。根据你的机器性能来调整这个值。

### `angularDefinitionProvider.maxCachedFiles`
- **描述**：最大缓存文件数。
- **默认值**：100
- **说明**：这个设置控制可以同时缓存多少个已解析的文件信息。增加这个值可以提高性能，但会占用更多内存。

### `angularHelper.enableLogging`
- **描述**：是否启用普通日志记录。
- **默认值**：true
- **说明**：启用此选项后，扩展会在输出面板中记录详细的运行日志。这对于诊断问题很有用，但可能会略微影响性能。

### `angularHelper.rootDirAliases`
- **描述**：根目录别名映射。
- **默认值**：  ```json
  {
    "__ROOT__": "./",
    "__PUBLIC__": "./Public"
  }  ```
- **说明**：这个设置允许你定义自定义的根目录别名。键是别名，值是相对于工作区根目录的路径。

### `angularHelper.ignoreScriptPatterns`
- **描述**：要忽略的脚本文件模式。
- **默认值**：`["*.min.js", "http://*", "https://*"]`
- **说明**：这个设置允许你指定应该被忽略的脚本文件模式。支持通配符 *。默认情况下，它会忽略压缩的 JS 文件和网络 JS 文件。

### 如何修改配置

1. 在 VS Code 中，打开设置（File > Preferences > Settings）。
2. 搜索 "Angular Helper" 或 "Angular Definition Provider"。
3. 找到相应的设置项并修改其值。
4. 某些设置可能需要重新加载窗口才能生效。

注意：这些设置会影响扩展的性能和行为。如果你不确定如何设置，建议保持默认值或稍作调整后观察效果。

## 注意事项

- 该扩展需要 Visual Studio Code 1.93.0 或更高版本。
- 该扩展专门针对 AngularJS (Angular 1.x) 项目优化。
- 为了获得最佳性能，建议在较小的项目或限定的文件范围内使用。
- 首次使用时，扩展需要一定时间来解析项目文件。之后的使用将会更快。

## 性能优化

Angular 助手扩展实现了以下高级优化策略，以显著提高性能和用户体验：

### HTML 和 JS 文件关联索引与智能解析

1. **优先建立关联索引**：
   - 在项目初始化时，首先扫描所有 HTML 文件。
   - 分析 HTML 文件中的脚本引用，如 `<script>` 标签。
   - 建立 HTML 文件到 JS 文件的映射，以及 JS 文件到 HTML 文件的反向映射。
   - 对于没有外部脚本引用的 HTML 文件，标记为内部实现 JS 逻辑。

2. **基于索引的异步解析**：
   - 根据建立的索引，异步执行每个 HTML 文件的解析。
   - 优先解析当前打开的文件：
     - 如果打开的是 HTML 文件，立即解析该文件及其关联的 JS 文件。
     - 如果打开的是 JS 文件，检查是否有关联的 HTML 文件，并优先解析这些文件。

3. **智能查找和跳转**：
   - 在 HTML 文件中查找 JS 定义时，直接定位到关联的 JS 文件（如果存在）进行解析。
   - 在 JS 文件中进行查找时，检查是否有对应的 HTML 文件关联，如果有，则包含相关 HTML 文件在解析范围内。
   - 对于内部实现 JS 逻辑的 HTML 文件，直接在 HTML 文件内进行解析。

4. **实时索引更新**：
   - 监听文件变化事件，特别是 HTML 文件的变化。
   - 当检测到 HTML 文件变化时，重新分析该文件的脚本引用，并更新索引。

5. **默认忽略文件夹**：
   - 为了提高性能，扩展默认忽略以下文件夹：
     - .git
     - .history
     - .idea
     - .vscode
     - doc
   - 这些文件夹通常包含版本控制和编辑器配置文件，与 AngularJS 代码无关。

这种优化方法带来以下好处：

- 显著提高了初始化速度和响应性，特别是对于大型项目。
- 通过优先处理当前打开的文件，提供了更快的即时反馈。
- 减少了不必要的文件解析，降低了内存和 CPU 使用。
- 准确处理了不同类型的 AngularJS 项目结构，包括外部 JS 文件和内联 JS 逻辑。
- 通过忽略无关文件夹，进一步提高了性能和降低了资源消耗。
